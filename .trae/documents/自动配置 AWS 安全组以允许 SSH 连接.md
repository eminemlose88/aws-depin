出现 "SSH Connection Failed: timed out" 错误通常是因为 AWS 的 **安全组 (Security Group)** 防火墙默认拦截了 SSH 连接（端口 22）。

目前的 `launch_instance` 代码使用了 AWS 的默认安全组，而默认安全组通常不开放外部访问。

### 修复方案

我将修改 `logic.py`，增加自动配置防火墙的逻辑：

1.  **新增 `ensure_security_group` 函数**：
    *   检查是否存在名为 `DePIN-Launcher-SG` 的安全组。
    *   如果不存在，则创建一个，并自动添加 **允许 SSH (端口 22) 来自任意 IP (0.0.0.0/0)** 的规则。

2.  **更新 `launch_instance` 函数**：
    *   在启动实例时，自动绑定这个开放了 SSH 权限的安全组。

这样，以后启动的所有新实例都将默认允许 SSH 连接，从而解决超时问题。

**注意**：此修复仅对**新启动**的实例生效。对于已经启动且无法连接的实例，建议将其销毁并重新部署（或者手动在 AWS 控制台修改其安全组）。
